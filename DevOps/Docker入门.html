Docker入门 ​

Docker介绍 ​

学习原因 ​
  • 解决传统部署依赖难以管理、环境不一致
  • Docker一次构建，不同环境多次运行
使用场景 ​

  • 开发、测试、生产环境统一

  • 持续集成部署

  • 资源占用少

基础概念 ​

  • 镜像（Image）：应用及其运行所需环境的打包模板
  • 容器（Container）：镜像运行起来后的实例
  • DockerHub：官方镜像仓库
  • 仓库（registry）：存放镜像，例如Registry、Harbor
  • 数据卷（Volume）：用于持久化和共享数据
  • 网络（Network）：管理容器之间的通信

安装 ​

bash
# 设置仓库
sudo dnf -y install dnf-plugins-core
sudo dnf config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
# 安装Engine
sudo dnf install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
sudo systemctl enable --now docker
# 安装DockerCompose
# 查看最新版本号：https://github.com/docker/compose/releases
DOCKER_COMPOSE_VERSION=<版本号>  # 例如 DOCKER_COMPOSE_VERSION=v2.27.1
sudo curl -L "https://github.com/docker/compose/releases/download/${DOCKER_COMPOSE_VERSION}/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose
docker-compose --version

配置镜像加速

bash
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<EOF
{
  "registry-mirrors": ["https://<your-mirror>.mirror.aliyuncs.com"]
}
EOF
sudo systemctl restart docker

常用命令 ​

镜像相关 ​

bash
docker pull nginx # 下载镜像
docker images # 查看镜像
docker rmi nginx # 删除镜像

容器相关 ​

bash
docker run -d -p 8080:80 --name web nginx # 运行容器
docker ps # 查看运行中的容器
docker stop web # 停止容器
docker exec -it web bash # 进入容器
docker logs web # 查看容器日志

容器调试 ​

bash
docker exec -it web /bin/sh
docker logs -f web
docker inspect web

网络模式 ​

Bridge：默认模式，会为每个容器分配、设置IP，并将容器连接到一个docker0虚拟网桥，通过docker0网桥以及iptables与宿主机通信

Host：容器将不会虚拟自己的网卡，使用宿主机的IP和端口

None：关闭容器网络功能，与宿主机、其他容器不联通

Overlay：用于Swarm集群，支持跨主机通信；多用于分布式系统和微服务

Macvlan：高级网络模式，容器拥有独立的Mac和IP，直接接入物理网络

IPvlan：容器共享Mac地址，但拥有独立IP

自定义：灵活配置子网、网关，支持DNS解析

Dockerfile ​

书写Dockerfile ​
dockerfile
FROM python:3.9-slim
WORKDIR /app
COPY . .
RUN pip install -r requirements.txt
CMD ["python", "app.py"]
构建镜像 ​
bash
docker build -t my-flask-app .
运行容器 ​
bash
docker run -d -p 5000:5000 my-flask-app

数据卷 ​

容器内的数据在容器删除后会丢失，可以使用Volume持久化数据

bash
docker run -v /home/web:/etc/nginx web

Docker Compose ​

可以同时管理多个容器，将他们都定义在1个docker-compose.yml文件中

yaml
version: '3'
services:
  web:
    image: nginx
    ports:
      - "80:80"
  redis:
    image: redis
使用方式 ​
bash
docker compose up -d
docker compose down

镜像优化 ​

镜像越小，构建越快，推送、拉取越方便

减少不必要的依赖和漏洞，提升安全性

多阶段构建 ​
bash
# 构建阶段
FROM node:18-alpine AS build
WORKDIR /app
COPY . .
RUN npm install && npm run build

# 运行阶段
FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
其他优化技巧 ​
  • 使用精简基础镜像(alpine等)
  • 合理使用.dockerignore排除无关文件
  • 合并RUN命令减少层数
  • 清理缓存文件：apt-get clean && rm -rf /var/lib/apt/lists/*

私有镜像仓库和镜像管理Harbor ​

  • 镜像权限管理
  • 签名和漏洞扫描
  • 与CI/CD、Webhook集成
