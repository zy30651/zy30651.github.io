Kubernetes 运维管理 ​

Helm 包管理器 ​

k8s 的生态中，所有的应用都去自己构建比较麻烦，而且是重复性工作；这些可以使用 helm

是查找、分享、使用软件构建 k8s 的最优方式

helm 管理名为：chart 的 k8s 包的工具，可以做以下事情：

  • 从头开始创建新的 chart
  • 将 chart 打包程归档的文件（tgz）
  • 与存储 chart 的仓库进行交互
  • 与现有的 k8s 集群中安装和卸载 chart
  • 管理与 Htlm 一起安装的 chart 的发布周期

Helm 架构 ​

3 个概念 ​

  1. chart 创建 k8s 应用程序所必须的一组信息

  2. config 包含可以合并到打包的 chart 中的配置信息，用于创建 1 个可发布的对象

  3. release 是一个与特定配置相结合的 chart 的运行实例

安装 Helm ​

  1. 下载二进制文件
  2. 解压 helm
  3. 将解压目录下的 heml 程序移动到/usr/local/bin/heml
  4. 添加阿里云 helm 仓库
bash
wget https://get.helm.sh/helm-v3.2.3-linux-amd64.tar.gz
tar -zxvf helm-v3.2.3-linux-amd64.tar.gz
# 添加仓库
helm repo add ingress-nginx https://kubernetes.github.io/ingress-nginx
# 查看仓库list
helm repo list
# 搜索仓库
helm search repo redis
helm search hub redis
# 已安装查询
helm list
# 加上命名空间
helm list -n ingress-nginx

Helm 常用命令 ​

bash
# 列出、增加、更新、删除仓库
helm repo
# 使用关键词搜索chart
helm search
# 拉取远程仓库的chart到本地
helm pull
# 在本地创建新的chart
helm create
# 管理chart依赖
helm dependency
# 安装chart
helm install
# 列出所有release
helm list
# 检查chart配置是否有误
helm lint
# 打包本地chart
helm package
# 回滚release到历史版本
helm rollback
# 卸载release
helm uninstall
# 升级release
helm upgrade

|  |  |--test-connection.yaml #
|-- values.yaml # 定义chart模板中的自定义配置的默认值，可以在

bash

helm history redis -n redis
# 1为revision的编号
helm rollback redis 1 -n redis
# helm卸载redis
helm delete redis -n redis

ELK 日志管理 ​

ELK 日志组成 ​

k8s 底层使用的容器引擎是 Docker，Docker 会有一个容器统一的日志文件，放在/var/log/containers

Filebeat：go 语言编写的轻量级日志收集工具，专门负责日志收集，可以监听每台机器下的日志文件，如果有日志，就收集后传给 Logstash；

Logstash：负责数据收集，数据清洗，不直接使用 Logstash，因为 Logstash 是 java 写的，资源损耗较重；Logstash 收到数据后，会发送给 Elasticsearch;

如果数据量过大，Logstash 可能消费不过来，可以接入中间件比如 kafka。

Elasticsearch：是文档数据库，专注与搜索场景。提供了全文搜索，精确匹配；

Kibana：负责数据展示，可以做数据检索和图表
