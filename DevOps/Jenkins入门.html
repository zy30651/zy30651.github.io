Jenkins入门 ​

Jenkins简介 ​

  • 持续集成（CI）：每次提交代码后自动构建、测试
  • 持续部署（CD）：自动将构建好的包部署到测试或生产环境
  • 自动化任务调度：定时执行构建、发布脚本

与Git、Docker、Kubernetes等无缝集成

安装配置 ​

bash
docker run -d \
  --name jenkins \
  -p 8080:8080 -p 50000:50000 \
  -v jenkins_home:/var/jenkins_home \
  jenkins/jenkins:lts

访问 http://localhost:8080，输入初始化密码（通过容器日志或 jenkins_home/secrets/initialAdminPassword 获取），即可进入 Jenkins 界面。

创建任务 ​

  • 自由风格 ​
    1. 新建任务-自由风格
    2. 配置构建逻辑
      1. 选择Git，填入Git地址，选择密钥
      2. 构建触发器：配置定时构建/SCM轮询
      3. 添加Shell
  • Pipeline流水线 ​

    将构建流程以脚本方式定义在Jenkinsfile中，支持版本控制和可视化

    groovy
    pipeline {
    agent any

    stages {
        stage('拉取代码') {
            steps {
                git 'https://github.com/user/project.git'
            }
        }
        stage('构建') {
            steps {
                sh 'echo Build started...'
            }
        }
        stage('测试') {
            steps {
                sh 'echo Running tests...'
            }
        }
    }
}

Pipeline语法 ​

是一种定义和自动化软件构建、测试、部署流程的脚本语言。使得流水线逻辑转为代码版本化，通常保存在Jenkinsfile文件中

有2种写法

声明式：结构清晰，语法规范，适合大多数用户；适用于初学和团队协作

脚本式：灵活强大，基于Groovy脚本，推荐高级用户

声明式语法 ​

  • 基础结构

    groovy
    pipeline {
    agent any

    stages {
        stage('Build') {
            steps {
                echo '开始构建'
            }
        }
        stage('Test') {
            steps {
                echo '执行测试'
            }
        }
        stage('Deploy') {
            steps {
                echo '部署应用'
            }
        }
    }

    post {
        success {
            echo '构建成功 🎉'
        }
        failure {
            echo '构建失败 ❌'
        }
    }
}

  • 关键语法

    pipeline：整个pipeline的起始标记

    agent：指定构建在那个节点上运行

    stages：包含多个阶段的集合

    stage：定义一个阶段，例如编译、测试、部署

    steps：该阶段要执行的命令

    post：构建结束后的清理、通知等动作

  • 语法特性

    • 环境变量

      groovy

  DEPLOY_ENV = 'prod'

    • 参数化构建

  string(name: 'BRANCH', defaultValue: 'main', description: '要构建的分支')

    • 工具安装

      • 通过Jenkins UI安装

      • 通过Jenkinsfile或Pipeline

        groovy

    maven 'Maven 3.8.6'

}

    • 并行执行

      groovy
      stage('Parallel Tasks') {
    parallel {
        stage('Task A') {
            steps {
                echo 'Running Task A'
            }
        }
        stage('Task B') {
            steps {
                echo 'Running Task B'
            }
        }
    }
}
