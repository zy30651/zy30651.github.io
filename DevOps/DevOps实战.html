完整流程 ​

DevOps：让技术团队、运维、测试等团队实现一体式流程自动化

CICD：

  • 持续集成(CI)：从编码、编译、测试、发布的完整自动化流程
  • 持续交付(CD)：包含持续集成，并且增加将项目部署到对应环境的自动化流程

  • 使用 helm 安装 jenkins 需要需改 rbac，否则在 jenkins 内的 k8s 插件无法新增 clouds

    yaml
    rbac:
  # 启用 RBAC 权限规则
  rules:
    - apiGroups: [""]
      resources: ["pods", "pods/exec", "pods/log", "secrets"]
      verbs: ["create", "delete", "get", "list", "watch", "patch"]
    - apiGroups: ["extensions", "apps"]
      resources: ["deployments"]
      verbs: ["create", "delete", "get", "list", "watch", "patch"]

serviceAccount:
  create: true # 确保创建 ServiceAccount
  name: "jenkins" # 明确指定 ServiceAccount 名称
  automountServiceAccountToken: true # 必须启用，否则无法挂载 Token

      2. Opened Merge Request Events：有合并请求时触发

      bash
      # 临时安装Gitea
cd /usr/local/src
wget -O gitea https://dl.gitea.com/gitea/1.23.1/gitea-1.23.1-linux-amd64
chmod +x gitea
groupadd --system git
adduser \
   --system \
   --shell /bin/bash \
   --comment 'Git Version Control' \
   --gid git \
   --home-dir /home/git \
   --create-home \
   git

mkdir -p /var/lib/gitea/{custom,data,log}
chown -R git:git /var/lib/gitea/
chmod -R 750 /var/lib/gitea/
mkdir /etc/gitea
chown root:git /etc/gitea
chmod 770 /etc/gitea
export GITEA_WORK_DIR=/var/lib/gitea/
cp gitea /usr/local/bin/gitea
# 此处有创建启动文件，已忽略
sudo vim /etc/systemd/system/gitea.service

sudo systemctl enable gitea
sudo systemctl start gitea
systemctl restart gitea
# 配置选项将写入以下位置: /etc/gitea/app.ini
