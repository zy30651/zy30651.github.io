安装和配置 ​

1. 前置 ​

jdk 1.8+、Maven 3.2+

bash
java -version 
mvn -version

2. 下载二进制包 ​

bash
# 安装方法1-git
git clone https://github.com/alibaba/nacos.git
cd nacos/
mvn -Prelease-nacos -Dmaven.test.skip=true clean install -U
ls -al distribution/target/

// change the $version to your actual path
cd distribution/target/nacos-server-$version/nacos/bin

# 二进制安装
# https://github.com/alibaba/nacos/releases
unzip nacos-server-$version.zip 或者 tar -xvf nacos-server-$version.tar.gz
cd nacos/bin

3. 启动服务器 ​

bash
# mac/linux/Unix
sh startup.sh -m standalone
# windows
startup.cmd -m standalone
# docker 降低一些配置
docker run --name nacos-standalone -e MODE=standalone -e JVM_XMS=512m -e JVM_XMX=512m -e JVM_XMN=256m -p 8848:8848 -d nacos/nacos-server:latest
# 访问
http://127.0.0.1:8848/nacos/index.html

注：Nacos的运行需要以至少2C4g60g*3的机器配置下运行。

4. Python 服务注册&发现和配置管理 ​

服务注册

bash
NACOS_URL = os.getenv(
    "NACOS_URL", "http://localhost:8848/nacos/v1/ns/instance")
SERVICE_NAME = "auth-service"
SERVICE_IP = socket.gethostbyname(socket.gethostname())
SERVICE_PORT = 5002
NAMESPACE = "python"

# 发送到Nacos服务注册接口
def register_service():
    payload = {
        "serviceName": SERVICE_NAME,
        "ip": SERVICE_IP,
        "port": SERVICE_PORT,
        "namespaceId": NAMESPACE,
    }
    response = requests.post(f"{NACOS_URL}", params=payload)

# 每5秒发送一次心跳
def send_heartbeat():
    while True:
        payload = {
            "serviceName": SERVICE_NAME,
            "ip": SERVICE_IP,
            "port": SERVICE_PORT,
            "namespaceId": NAMESPACE,
        }
        response = requests.put(f"{NACOS_URL}/beat", params=payload)
        time.sleep(5)
# 服务启动类
if __name__ == '__main__':
    register_service()
    heartbeat_thread = threading.Thread(target=send_heartbeat)
    heartbeat_thread.daemon = True
    heartbeat_thread.start()
    app.run(port=SERVICE_PORT)

配置中心 ​

py
import json
import logging
import nacos
from playhouse.pool import PooledMySQLDatabase
from playhouse.shortcuts import ReconnectMixin

NACOS = {
    "host": "localhost",
    "port": 8848,
    "namespace_id": "4ea66ceb-fa1c-4287-8bae-d0cda3959f10",
    "group": "dev",
    "data_id": "user-srv.json"
}
client = nacos.NacosClient(f"{NACOS['host']}:{NACOS['port']}", namespace=NACOS["namespace_id"])
data = client.get_config(NACOS['data_id'], NACOS['group'])
json_data = json.loads(data)
logging.info(json_data)
def update_cfg(args):
    print("配置发生变化")
    print(args)
# 服务相关配置
SERVICE_NAME = json_data['name']
# 使用peewee的连接池，使用ReconnectMixin，防止出现链接断开查询失败
class ReconnectMysqlDatabase(ReconnectMixin, PooledMySQLDatabase):
    pass
db = ReconnectMysqlDatabase(database=json_data['mysql']['db'],
                            host=json_data['mysql']['host'],
                            port=json_data['mysql']['port'],
                            user=json_data['mysql']['user'],
                            password=json_data['mysql']['password'])

Nacos的基本使用 ​

1.命名空间 ​

默认public，可以将所有的配置文件统一放在这里；也可以分开存放

命名空间一般用来区分微服务；一个微服务一个命名空间，比如users包含user-srv、user-web

2.组 ​

现在能够隔离微服务；但是不同微服务的开发、测试、生产环境如何区别;

3.Data ID 配置集 ​

一个配置集就是一个文件；实际上可以更灵活；把一个文件的各个部分分开存储到不同配置集
